## Custom Project

### Dataset
Using the original dataset of reports, errors and latency. With additional data points added.

### Signals
Success rate (instead of error rate) and latency health report.

### Experiments
First modified the error rate to display as a percentage instead of a decimal for readability. Then modified the signals to catch success rate and engineered features to provide a new collumn output to show health report of the latency because seeing raw numbers may not be human-friendly.

### Results
The health reports for the original data was all "healthy", so I added additional data points to generate "warning" and "critical" health reports.

### Interpretation
The new output is more human-friendly because it displays the success rate in a easier to read format and the latency is categorized in three levels for humans to understand if there is a concern or not.
